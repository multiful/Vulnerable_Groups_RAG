■ 목적 및 배경

서울시 「고립·은둔 청년 실태조사」(2022, n=5,513) 분석 결과,
서울시 청년의 8.82%가 고립·은둔 상태로 분류됩니다.

  · 고립군 청년 50.2% — "지난주 일을 하지 않음" (비고립군 9.8%, Cohen's h ≈ 0.94)
  · 고립군 청년 62.6% — "대면교류 전혀 없음"   (비고립군 8.0%, Cohen's h ≈ 1.25)
  · 은둔·고립 원인 1위 — 취업 어려움 (해당 응답자의 50.6%)

보건복지부 「고립·은둔 청년 실태조사」(2023, n=21,360) 추가 확인:
  · "진로·미래를 생각하면 막막하다" 그렇다 응답: 74.7%
  · "일을 할 수 없을 것 같다" 그렇다 응답: 43.4%

이처럼 취약 청년은 노동·관계·정서가 동시에 무너지는 다층적 단절을
경험하지만, 현행 공공 서비스는 세 가지 구조적 한계를 갖습니다.

  ① 사용자 반영성 부재
     개인의 위험 단계를 반영하지 않은 획일적 자격증 목록만 제공.
     최고위험군(5단계) 청년에게는 낮은 난이도부터, 안정권(1단계) 청년에게는
     기사·기술사 도전을 유도하는 것이 적합하지만 현재 플랫폼은 이를 구분하지 않습니다.

  ② 채널 분리
     자격증(Q-Net), 채용(워크넷), 훈련(Work24), 심리지원이 별도 플랫폼으로 분산.
     정보 탐색 비용이 높아 취약 청년이 중도 포기하는 구조입니다.

  ③ 단발성 지원
     일회성 상담·인증서 발급에 그쳐 자격증 취득 후 "다음 단계"를 안내하는
     연계형 지원이 없습니다.

DIDIM(디딤)은 이 세 가지 한계를 동시에 해소하기 위해 개발하였습니다.
"디딤"은 "첫 발을 내디딘다"는 뜻으로, 취약 청년의 노동 재진입 첫 걸음을
지원한다는 서비스 철학을 담았습니다.

■ 핵심 기능 및 특징

DIDIM은 진단(Diagnose) → 매칭(Match) → 경로(Path) → 근거(Evidence)
4단계 파이프라인으로 동작합니다.

【기능 1】 위험군 5단계 자동 진단 (12문항)
  · 12문항: 관계망 4문항 + 노동·경제 4문항 + 심리·안전 4문항
  · Cohen's h 변별력 상위 8문항 선별 + 보강 4문항
  · 가중 합산 점수 → 경계값(12.71 / 18.21 / 29.62 / 35.55) 기준 5단계 분류
  · 검증: 시뮬레이션 ROC-AUC 0.9915

    단계   의미              경계 점수
    ─────────────────────────────────────
    1단계  취업 안정권       ~ 12.71
    2단계  불안정 근접       ~ 18.21
    3단계  경계선 위험군     ~ 29.62  ← 양 집단 접점, 서비스 효용 극대화
    4단계  고위험군          ~ 35.55
    5단계  최고위험군        35.55 ~

【기능 2】 도메인·직무 기반 자격증 추천
  · 관심 도메인 9개 대분류 / 44개 세부 라벨
  · 위험군 단계별 가중치 적용 (단계가 높을수록 낮은 난이도 우선)
  · 자격증 1,290건 마스터 DB 기반 매칭

【기능 3】 DAG 기반 단계형 취득 로드맵
  · 자격증 간 선수 관계를 방향 비순환 그래프(DAG)로 구성
  · 예시 (IT 도메인):
    정보처리기능사 → 컴퓨터활용능력2급 → 정보처리산업기사 → 정보처리기사 → 정보관리기술사

【기능 4】 RAG 기반 추천 근거 제시
  · Q-Net 공식 문서(PDF·HTML) 기반 근거 스니펫 실시간 제공
  · 환각 방지: 근거 외 임의 수치·날짜 생성 금지

【기능 5】 실행 정보 원스톱 연동
  · 시험 일정·접수 일정 (Q-Net API 실시간)
  · 관련 채용공고 (워크넷 API 실시간)
  · 훈련과정 (Work24 + 과정평가형 API 실시간)
  · 주변 인프라 지도 (일자리카페·건강증진센터·훈련기관)
  · 오늘의 한 행동 추천 (단계별 맞춤 소행동 카드)

  활용 범위

  · 확장: 전국 광역시 공공데이터 연동 시 전국 서비스 확대 가능
  · 타 취약계층(장애인·경력단절여성 등) 데이터 연동 시 동일 파이프라인 재사용

  가공 방식
  · CSV → canonical entity·relation·candidate row 정규화 (structured lane)
  · PDF·HTML → 파싱·청킹·임베딩 → pgvector 인덱스 (evidence lane)
  · 두 lane 분리: 추천 정확도(structured)와 설명 근거(evidence) 독립 유지

■ 활용된 AI 기술 목록

  AI 기술                     | 활용 목적
  ──────────────────────────────────────────────────────────────
  통계 모델 (SHAP 분석)       | 위험군 5단계 자동 진단
  벡터 임베딩 (OpenAI)        | 문서 의미 검색 (RAG)

  Kakao REST API (지오코딩)   | 주소 → 위경도 자동 변환

────────────────────────────────────────────────────────────────
■ AI 기술 1 — 통계 모델 기반 위험군 진단

  기술: SHAP + 가중 합산 분류

  · 서울시 실태조사 138문항 중 Cohen's h 변별력 상위 8문항 + 보강 4문항
  · SHAP 기반 취업 예측 기여도 가중치 (R² = 0.72)
  · 시뮬레이션 검증: ROC-AUC 0.9915

  지속성:
  · 신규 실태조사 데이터 공개 시 경계값 재계산으로 자동 업데이트
  · 문항 가중치 조정만으로 전국 데이터 확장 적용 가능

────────────────────────────────────────────────────────────────
■ AI 기술 2 — RAG (Retrieval-Augmented Generation)

  기술: 벡터 임베딩 + pgvector + LangChain + 생성형 AI

  파이프라인:
  [오프라인] PDF·HTML 파싱 → 청킹 → 임베딩(OpenAI text-embedding-3-small)
             → PostgreSQL + pgvector HNSW 인덱스

  [온라인]  질의 → 벡터 검색(Top-K) → 스니펫 추출
             → LangChain 조립 → LLM 응답 → 화면 표시

  환각 방지:
  · 근거 외 임의 정보 생성 금지
  · 근거 0건 → "설명 근거 없음" 명시

  지속성:
  · Hash-Incremental: 원본 파일 해시 변경분만 재임베딩
  · 신규 공공데이터 즉시 파이프라인 추가 가능

────────────────────────────────────────────────────────────────
■ AI 기술 3 — 지식 그래프 + DAG 로드맵

  기술: 방향 비순환 그래프(DAG) 순회

  · 자격증 간 선수·후수 관계 엣지 정의
  · BFS/DFS 순회로 현재 단계에서 도달 가능한 경로만 추출
  · 순환 없는 논리적 취득 경로 보장

  확장:
  · NCS 능력단위 연동으로 직무 기반 경로 고도화 예정

■ 창업(사업) 추진 의지

DBMs팀은 DIDIM을 공모전 출품작이 아닌 실제 운영 서비스로 구축하였습니다.

UptimeRobot으로 5분 간격 상태 모니터링을 운영하고 있습니다.
